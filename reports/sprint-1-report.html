1. Sprint Overview

Sprint 1 delivered a complete, production-ready companion website for Board of Innovation's webinar "How AI is Disrupting Business Models," presented by Laura Stevens (Managing Director, BOI). The site is a single-page editorial scroll with seven narrative sections, 16 integrated presentation slides, dark/light mode, and a full lightbox slide viewer.

Objective

Build and ship a magazine-quality companion site that transforms a 45-minute webinar into a structured, interactive reading experience. Pure static HTML, CSS, and vanilla JS. No frameworks, no build tools. Deployable to Cloudflare Pages.

2.2 Design System

The Creative Director delivered a comprehensive design system as design-system.css (1,276 lines), establishing:

  • Color palette: Dark mode default (#0a0a0a base, #4ecdc4 teal accent). Light mode overrides (#f5f5f5 base, #2ba89e darker teal). All colors managed through 20+ CSS custom properties.
  • Typography: Inter (body/UI) and Playfair Display (headings/quotes). Fluid sizing via clamp() across hero, section titles, pull quotes, and body text.
  • Spacing: 4px-base scale from --space-1 (4px) through --space-10 (128px).
  • Components: Slide cards, case study cards, overview cards, shift cards, index cards, track cards, pull quotes, full-bleed quote dividers, "So What?" callouts, Q&A items, CTA buttons.
  • Shadows and elevation: Theme-aware card shadow system (--card-shadow, --card-hover-shadow).
  • Radius and z-index tokens: Consistent border-radius scale and layered z-index for topbar, sidebar, overlays, and menus.

2.3 Interactive Features

  • Theme toggle: Dark/light switch in the top bar with localStorage persistence and smooth CSS transitions.
  • Scroll spy: IntersectionObserver-based active state on both sidebar and mobile menu navigation. URL hash updated via history.replaceState.
  • Lightbox slide viewer: Click any of 16 slides to open a fullscreen viewer. Previous/next navigation, keyboard support (arrows, Escape), and focus trapping.
  • Mobile menu: Slide-in panel with overlay backdrop. Focus trap for keyboard users. Escape to close. Focus restored to trigger element on close.
  • Quick Take toggle: Shows 8 of 29 bullets by default. "Show all 29 bullets" button reveals the remainder.
  • Smooth scroll: CSS scroll-behavior: smooth combined with JS offset calculation for the fixed top bar.
  • Scroll restoration: Resets to top on page load for a consistent first impression.

3.2 Architecture Decisions

  • Single-page scroll: The webinar presents one linear argument across seven chapters. A single-page format mirrors this structure and encourages continuous reading.
  • CSS custom properties: 30+ variables for colors, spacing, typography, shadows, and transitions. Complete theme switching with a single data-theme attribute.
  • IntersectionObserver for scroll spy: More performant than scroll event listeners. Avoids layout thrashing and jank.
  • Vanilla JS in IIFE: All JavaScript in an immediately-invoked function expression with 'use strict'. No global variables. No frameworks or libraries.
  • No build tools: Plain HTML, CSS, JS. No bundler, no preprocessor. Deployable as-is to any static host.
  • Glassmorphism top bar: Fixed header with backdrop-filter: blur(20px) and semi-transparent background, adapting to both themes.

3.3 Accessibility

  • Skip-to-content link as first element in <body>
  • ARIA attributes on all interactive elements: aria-label, aria-expanded, role="dialog", aria-modal="true"
  • Semantic HTML5 landmarks: <header>, <nav>, <aside>, <main>, <section>, <footer>
  • Focus trapping in mobile menu and lightbox; focus restoration on close
  • Keyboard navigation: arrows and Escape in lightbox, Enter/Space on slide cards, Tab cycling
  • @media (prefers-reduced-motion: reduce) disables all animations and transitions
  • :focus-visible outlines; suppressed on mouse click via :focus:not(:focus-visible)
  • Descriptive alt text on all 14 inline slide images
  • .sr-only utility class for screen-reader-only text
  • Print styles: hides interactive elements, resets colors, adds page-break rules

3.4 Performance

  • All slide images use loading="lazy"
  • Zero JavaScript dependencies; total JS is 422 lines
  • No build step required
  • Window resize handler debounced at 100ms
  • Single IntersectionObserver instance for all sections

5.2 Issues Found and Resolved

Fixed During Sprint

  1. Slide card keyboard accessibility (Medium): Slide cards were clickable but not keyboard-accessible. Added tabindex="0", role="button", aria-label, and Enter/Space key handlers. Fixed by QA Reviewer in script.js.
  2. Missing skip-to-content link (Critical, WCAG 2.1 Level A): Added as first element in body, visually hidden by default, appears on keyboard focus. Fixed by UX Designer in index.html.
  3. Missing aria-expanded on burger button: Added to communicate initial menu state to screen readers. Fixed by UX Designer in index.html.

Documented for Future Sprints

  1. Heading hierarchy skip (Medium): Card headings use <h4> without intervening <h3> in some sections. Not a critical blocker.
  2. Section number double-opacity (Low): .section-number applies both semi-transparent color and opacity. Confirmed as intentional decorative styling.
  3. Touch targets slightly below 44px (Minor): Burger and theme toggle are 40x40px. Lightbox nav buttons drop to 36px at 480px.

5.3 Final Assessment

Verdict: Production-ready. All 11 QA checks pass. Three issues were found and fixed during the sprint. Three minor items documented for future improvement. No critical blockers remain.

7. Recommendations for Future Sprints

7.1 Features Not Implemented

  • Animated scroll reveals: Sections could fade in as the user scrolls. IntersectionObserver is already in use, so adding this would be straightforward.
  • Reading progress indicator: A thin accent bar at the viewport top showing scroll progress. Common in long-form editorial sites.
  • Reading time per section: The UX spec included per-section reading time estimates. These could be displayed as subtle labels on section headers.
  • Social sharing buttons: Share links for Twitter/X, LinkedIn, and email.
  • "Connect with Laura" CTA: The written-content spec included a LinkedIn CTA, but the URL was not available at build time.
  • AI Disruption Index self-assessment: An interactive version of the 8-factor framework where readers score their own business model vulnerability.

7.2 Known Limitations

  • Heading hierarchy: Card headings skip from <h2> to <h4> in some sections. Should be corrected for stricter WCAG compliance.
  • Touch targets: Burger, theme toggle (40px), and lightbox nav at 480px (36px) fall slightly below the 44px WCAG recommendation.
  • No scroll-margin-top: Native hash navigation does not account for the fixed top bar. One CSS property would fix this.
  • 3 unused slides: Slides 0003, 0005, 0006 are available but omitted as redundant. Could be added to the lightbox for completeness.
  • No favicon: A BOI-branded icon would complete the browser tab experience.
  • Burger aria-expanded not toggled dynamically: Set on page load but not updated when the menu opens or closes.

7.3 Potential Improvements

  • Open Graph meta tags: Add og:title, og:description, og:image for social sharing previews.
  • Structured data (JSON-LD): Schema.org markup for the webinar, speaker, and organization.
  • Analytics: Lightweight privacy-friendly analytics (Plausible, Fathom) for engagement tracking.
  • PDF export: A "Download as PDF" button leveraging the existing print styles.
  • B44 naming verification: Content uses "B44" consistently; the QA report notes written-content.md references "Bolt.new" for the same case study. Verify the canonical name.
